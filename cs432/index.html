Cloud Computing

CS432  ·  Sacred Heart University
Course Description  ·  Undergraduate Catalog

This course presents a number of cloud computing tools and technologies, including virtualization, web services, data analysis, and integration.

3 Credits
Prereq: CS-112 & CS-339
Hands-on Labs & Projects
Near Azure Foundations Certification Ready

Curriculum maps closely to AZ-900 exam domains — cloud concepts, Azure architecture, core services, security, governance, and cost management — positioning students to sit for the certification upon completion.

Built on a decade of firsthand Azure architecture experience
Course Topics
Cloud Concepts

IaaS, PaaS, SaaS service models. Public, private, and hybrid cloud. The shared responsibility model and the economic case for cloud adoption.

Azure Architecture

Regions, availability zones, resource groups, and subscriptions. Azure Resource Manager and the management hierarchy.

Compute & Storage

Virtual Machines, App Service, Azure Functions. Blob, File, Queue storage with redundancy options, tiers, and lifecycle management.

Identity & Security

Azure Active Directory, RBAC, MFA, and Conditional Access. Microsoft Defender for Cloud and Azure Policy for compliance governance. Key Vault for secrets, keys, and certificate management.

Networking

Virtual Networks, NSGs, VPN Gateway, and Load Balancer. Azure DNS, Application Gateway, and ExpressRoute connectivity options.

Cost Management

Azure pricing models, TCO Calculator, and budgets. Cost optimization strategies, resource tagging, and governance for billing control.

Web Services

Azure App Service, Azure Functions, and Logic Apps. Building, deploying, and integrating cloud-native web services and serverless workflows.

Governance

Azure Policy for compliance enforcement and guardrails. Defining, assigning, and evaluating policies to ensure resources stay within organizational standards.

Databases

Azure SQL Database for managed relational data, Elastic Pools for multi-tenant cost efficiency, and database monitoring for performance tuning, query insights, and alerting.

Monitoring

Azure Monitor for metrics and alerts, Log Analytics for querying telemetry with KQL, and Dashboards for unified operational visibility across resources and services.
